# 汽车销售管理系统 - 课程设计核心任务与要求总结
## 一、前提条件与组队规则（必看）
1. **组队限制**：可单独完成，或2人组队（最多2人），**严禁跨班组合**；组长需排在分工表第一位。
2. **选题规范**：同班内不可重复选题，若与他人题目一致，需采用**不同编程实现**（视为新题目）；需在**10月30日前完成选题确认**。
3. **分工要求**：按“需求分析、概念设计、逻辑设计、物理设计、编程实现、运行测试”6个环节分配任务，每个组员完成对应环节需在分工表中画√。

## 二、系统设计与实现核心任务
### 1. 基础功能（必备，占成绩核心）
- 数据库核心：必须包含**增、删、改、查4项基础功能**，查询需灵活设计（满足多样化需求）；额外需支持**数据导入、数据导出**。
- 扩展功能：结合汽车销售场景补充功能（如车辆库存管理、客户订单跟踪、销售统计、报表生成、员工管理等）。
- 数据规范：保证数据一致性；**不得使用自增字段作为主键**。

### 2. 技术选型（不限定，可自主选择）
- 后台DBMS：Oracle、MySQL、Microsoft SQL Server或其他数据库均可。
- 架构与前端：支持C/S、B/S（Web版）或移动端版（鼓励移动端开发），前端工具不限。

### 3. 界面与演示要求
- 界面：设计需美观、易用。
- 演示：需结合模拟/真实数据完成功能演示，答辩时准备**5分钟系统介绍**（含核心功能操作展示）。

## 三、课程设计报告编写要求（占成绩40%）
需严格按照参考模板结构编写（Word+PDF双版本），核心章节及汽车销售场景适配如下：
| 报告章节 | 核心内容（汽车销售管理系统适配） |
| --- | --- |
| 1. 系统开发背景 | 1.1 设计目的（如规范汽车销售流程、提高管理效率等）；1.2 设计内容（系统核心覆盖范围：车辆、客户、订单、库存等） |
| 2. 需求分析 | 2.1 产品描述（系统定位、解决的销售痛点）；2.2 产品功能（如车辆查询、订单创建、库存预警、客户管理等）；2.3 信息要求（明确系统需存储的核心数据）；2.4 数据字典（含数据项、数据结构、数据流图、数据存储、数据处理，示例：车辆编号、客户身份证号、订单编号等数据项的类型/含义/取值） |
| 3. 概念结构设计 | 3.1 分E-R模型图（如车辆信息、客户信息、订单信息、员工信息等单独ER图）；3.2 总E-R模型图（整合所有实体及关系） |
| 4. 逻辑结构设计 | 4.1 关系模型（如车辆表、客户表、订单表、库存表、员工表等，明确字段及主键）；4.2 关系模型优化；4.3 用户子模式设计；4.4 系统功能模块划分（如销售管理、库存管理、客户管理、统计报表等） |
| 5. 数据库物理设计 | 5.1 安全性和完整性设计（数据约束、权限控制）；5.2 索引设计（高频查询字段建立索引，如车辆型号、订单日期）；5.3 存储路径和存取方式设计 |
| 6. 数据库实施与维护 | 6.1 建立数据库及表（附SQL代码）；6.2 数据库关系图；6.3 数据插入与查询（附测试数据）；6.4 视图设计（如销售统计视图、库存预警视图）；6.5 存储过程设计（如订单创建、库存更新存储过程）；6.6 索引实现；6.7 用户界面设计（附截图）；6.8 备份与恢复方案 |
| 7. 心得体会 | 设计与实现过程中的问题、解决思路、总结感悟 |
| 8. 参考文献 | 至少包含数据库、编程相关教材或技术文档（如《数据库系统概论》等） |

## 四、成绩评定标准（总分100%）
| 评价类别 | 占比 | 核心要求 |
| --- | --- | --- |
| 课程设计报告 | 40% | 结构完整（覆盖上述所有章节）、内容详细、格式规范，包含相关技术介绍、测试报告、系统使用/安装说明 |
| 程序实现 | 30% | 运行正确、功能完善（含增删改查、报表、统计、输入输出功能）、工作量充分、界面友好、技术难度合理 |
| 数据库设计 | 30% | E-R图设计正确、逻辑结构规范化、物理设计合理（索引、存储等） |

## 五、需提交的材料与截止时间
### 1. 提交材料清单（缺一不可）
- 报告类：课程设计报告（Word版 + PDF版）。
- 数据类：SQL格式数据文件（含测试数据）。
- 代码类：系统完整源文件。
- 演示类：5分钟系统演示介绍材料（用于答辩）。
- 分工类：填写完整的小组分工表（标注组长及各成员完成环节）。

### 2. 提交时间与方式
- 截止时间：第20周周四晚9点前。
- 提交路径：所有材料打包发给各班学委，由学委统一刻录光盘提交给谢锐老师。
- 答辩安排：第21周进行（具体时间另行通知），需现场演示系统功能。

## 六、关键注意事项
1. 提前完成小组可获得“成绩×奖励系数”加分，建议尽早推进开发。
2. 文档格式需严格遵循规范（参考评分标准及报告模板），避免格式问题扣分。
3. 数据库设计需满足“无自增主键”“数据一致性”要求，查询功能需充分考虑用户实际需求（如按车辆品牌、价格区间、销售日期查询）。
4. 系统需结合汽车销售真实场景，确保功能实用性（如订单与库存联动、销售数据统计分析）。
